什麼是資料庫

我們的日常生活都會需要用資料庫來儲存我們所需的資料，將資料透過寫入資料庫來進行儲存；也可以透過資料查詢資料庫，資料庫再回傳給我們查詢所需的資料。

資料庫長甚麼樣

資料庫外觀會由「資料表、欄位、資料列」組成

  • 資料表 ( Table) ：是整體資料，包含了所有相關的資料
  • 欄位 ( Columns)：定義資料的屬性和類型 ( Data Type)
  • 資料列 ( Rows )：是橫向的資料集合，代表一筆完整的資料

資料庫運作流程

如上圖所示，例如我們要查詢沙發的資料

  • 當使用者端在網站上輸入查詢條件 ( 沙發 ) 後進行查詢

  • 中間會透過伺服器去跟資料庫要跟 ( 沙發 ) 相關所有有符合的資料後

  • 資料庫再透過中間的伺服器將查詢到 ( 沙發) 的資料回傳給使用者端

  • 使用者端就可以在網站頁面上查看到沙發的資料

什麼是 SQL

  • SQL 全名為 「 Structured Query Language 」
  • 中文為 「結構化查詢語言」

資料庫設計步驟

資料表建置步驟：

  1. 建立資料表 ( Table )，要建立那種資料
  2. 建立欄位 ( Columns )，有那些資料欄位
  3. 定義欄位資料類型 ( Data Type )，例如 字串、數字、日期等資料長度。

SQL 指令語法教學

可以透過 PG SQL 進行練習

  1. CREATE TABLE 建立資料表

    • 透過 CREATE TABLE 指令建立 users 資料表

    • () 括弧中建立所需的欄位跟定義欄位型別與長度

    • 最後再用 ； 分號結束。

  2. INSERT INTO，新增一筆資料到資料庫

    • 透過 INSERT INTO 新增資料到 users 資料表
    • 用 VALUES 與資料庫宣告要新增的資料
    • 要新增的資料，順序要跟上方 () 括號中資料順序一樣

  3. SELECT 查詢資料

    • 透過 SELECT 查詢資料， * 號為查詢資料表全部欄位資料。
    • FROM 指定要查詢的資料表。

    • 可以指定要查詢的欄位

  4. AS 新增別名

    • 如果希望欄位名稱查詢的結果是自己所訂的名稱，可以用 AS 新增別名。

    • 延伸使用，用年齡進行計算並設定欄位別名為 距離退休年數

  5. SQL 語法註解

    • 使用兩個 - 單行註解
    • 使用 /* SQL 指令 */ 多行註解

  6. WHERE 篩選資料

  7. 比較運算子

    • WHERE category = ‘3C’; 中 = 等於其他用法，如下

    • 情境：想知道哪些商品快沒貨了，有沒有庫存小於 50 的產品？

  8. AND 、OR 邏輯運算子

    • 使用 AND 進行判斷，多個條件同時要成立才可執行查詢
    • 使用 OR 進行判斷，其中一個條件成立就可以執行查詢
    • 情境 : 已上架的 3C 商品

    • 情境：下架或無庫存商品

  9. 集合與範圍運算子

    • BETWEEN : 在指定的範圍內
    • IN : 包含在列出的值中
    • NOT IN : 不包含在列出的值中
    • 情境 : 想幫家人買生日禮物，預算在 500~1000 元

    • 情境 : 找出特定類別商品

    • 情境 : 排除特定商品 (充電線、手機殼、螢幕保護貼);

  10. UPDATE 更新資料表

    • 情境 : 單筆刪除：刪除特定商品

  11. DELETE 刪除資料

    • 目前資料表的資料

    • 情境 : 單筆刪除：刪除特定商品
