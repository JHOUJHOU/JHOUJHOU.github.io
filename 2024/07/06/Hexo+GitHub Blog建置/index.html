建立自己的部落格 Hexo + GitHub Pages

發表於 2024-07-06 分類於 Blog build

前言

幾年有架設過 Hexo ，但是遇到一些問題，所以就來重新架設，也同時記錄學習架設過程。

什麼是 Hexo ?

引用 Hexo 網站中介紹

Hexo 是一個專門用來寫網誌的框架，有幾個特性 :

  1. 快速、簡單、易懂。
  2. 可以將靜態網頁一鍵部署到 GitHub Pages、Heroku 等。
  3. 支援 Markdown 格式。
  4. 種多的主題跟外掛套件可使用。

安裝所需前置作業

  1. 安裝 Node.js

    Hexo 需要使用 Node.js 環境，若原本沒有安裝，需要安裝起來。

    這裡是選擇安裝 16.16.0 LTS 版本， 安裝後可以使用 npm 來安裝。

  2. 安裝 Git

    用來將檔案部署到 GitHub 上，主要是用來做版本控制的程式。

  3. 安裝 VSCODE ( Visual Studio Code )

    使用編輯器，之後可以用來寫 Markdown 文章。

  4. 在 GitHub 上建立帳號及建立存儲庫 ( repository )

    username 需要修改成自己帳號為主的名稱。

Hexo 基本設定介紹

安裝 Hexo 安裝完成後，可以在專案資料夾中顯示下方資料夾結構 :

資料夾結構說明 :建立相對應的檔案

node_modules

所有需要安裝的套件都這在資料夾中

scaffolds

當要建立新的文章時，Hexo 會根據 scaffolds 資料夾建立檔案。

資料夾中有三個檔案，darft.md、page.md、post.md，分別對應草稿、頁面、文章。

soure

主要放原始檔案的地方，例如 Markdown 或HTML 檔案都放在這資料夾中。

如檔案或資料夾不想被編譯，可在前面加上 _ (下底線)來忽略，就不會被編譯或是複製到 public 資料夾中。

另外除了 _posts 資料夾，雖然有下底線，但是是不會被忽略的。

themes

放置主題的地方，官方有提供一個預設的主題 landscape，透過設定的主題及 _config.yml 設定編譯產生靜態網站檔案。

_config.yml

主要是網站的各種設定檔，例如: 主題設定、網站標題、網址等。

.gitignore

用來忽略特定檔案或資料夾，如不想被加入版控且上傳到 GitHub 的資料夾或檔案，都寫在 .gitignore，官方預設是忽略以下檔案及資料夾，基本上此檔案不會做修改，只需了解即可。

package.json

安裝的套件都儲存在 package.json 中，是拿來放置管理 npm 下載後的檔案。

package-lock.json

是 npm 5 才有新增的檔案，主要適用於紀錄當前安裝的套件版本，

為了確保安裝到正確的套件，基本上此檔案不會調整到，只有在新增、刪除及更新套件時，才會有異動。
